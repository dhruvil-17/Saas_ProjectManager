# 📘 SaaS Task Manager — Full Product & System Documentation

---

# 🧠 1. Product Overview

## 📌 Product Name

**SaaS Task Manager (Multi-tenant)**

## 🎯 Goal

A production-grade full-stack application where teams can:

* Create workspaces
* Collaborate on tasks
* Manage projects
* Receive notifications
* Upgrade via subscription plans

---

# 🧩 2. Core Concept

This is a **multi-tenant SaaS platform**.

👉 Key idea:

* One app
* Multiple organizations (workspaces)
* Each workspace has its own users, projects, and tasks

# 🧠 5. Architectural Pattern

## Layered Architecture:

```
Route → Controller → Service → Repository → Database
```

### Responsibilities:

* **Route** → defines endpoint
* **Controller** → handles request/response
* **Service** → business logic
* **Repository** → database queries

# 🔐 8. Authentication System

## Flow:

1. User signs up / logs in
2. Server issues:

   * Access Token (short-lived)
   * Refresh Token (stored securely)
3. Refresh token used to generate new access tokens

## Security:

* Refresh token stored in HTTP-only cookies
* Access token sent in headers
* Redis used for session tracking

---

# 🏢 9. Workspace System (Multi-Tenancy)

## Features:

* Create workspace
* Invite users
* Join workspace via invite
* Switch between workspaces

## Roles:

* Admin → full control
* Member → limited access

---

# 📋 10. Project & Task System

## Project:

* Created inside a workspace
* Groups tasks

## Task:

* Belongs to a project
* Assigned to a user
* Has status:

  * TODO
  * IN_PROGRESS
  * DONE

---

# 🔔 11. Notification System

## Events:

* User invited
* Task assigned

## Flow:

1. Event triggered
2. Message sent to RabbitMQ
3. Worker processes message
4. Email sent to user

---

# 💳 12. Billing System

## Features:

* Subscription plans (Free / Paid)
* Stripe checkout integration
* Webhook handling

## Flow:

1. User upgrades plan
2. Stripe processes payment
3. Webhook updates subscription status
4. Features unlocked
